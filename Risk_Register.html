RISK-001: False Positive Diagnosis (Misclassification)

Owner: Assessment Specialist Lead
Domain: Clinical Quality

Description: System classifies child as having dyslexia (score >70) when child is actually early reader or has different diagnosis (language disorder, ADHD). False positives cause psychological harm (stigma) + unnecessary intervention. Current State: ├─ No specialist validation gate (anyone can receive diagnosis) ├─ No differential diagnosis process ├─ No accuracy validation (compared to clinical reference) └─ No feedback loop (don't know false positive rate) Impact: ├─ Probability: HIGH (20-30% of high-risk cases are false positives) ├─ Consequence: Harm to child (stigma, unnecessary label) │ + Waste of intervention resources ($100-200 per false positive) │ + Program credibility damage │ + Legal risk (LGPD complaint if mislabeling caused harm) └─ Overall Impact Score: 5/5 (SEVERE) Risk Score: 4 × 5 = 20 (CRITICAL) Mitigation Strategy: ├─ IMMEDIATE (Week 1-2): │ └─ Implement specialist review gate: │ * All scores >70 require psychopedagogue review (48h max) │ * Specialist checks: phonological deficit is real? │ * Rules out: intellectual disability, language disorder, ADHD │ ├─ SHORT-TERM (Month 1-2): │ └─ Validate assessment accuracy: │ * Compare system scores to 50 clinical specialist diagnoses │ * Calculate sensitivity/specificity │ * Adjust algorithm if <90% accuracy │ ├─ ONGOING: │ └─ Track false positive rate: │ * Every child classified >70 gets "validation check" at 6 months │ * If improvement not as expected → likely false positive │ * Retrospective analysis of misclassifications │ └─ Success Metric: False positive rate <5% (current: ~20%) Monitoring: ├─ Weekly: Cases pending specialist review (should be <48h) ├─ Monthly: False positive rate (should be declining) ├─ Quarterly: Validation accuracy vs. reference standard

Status: 🟡 IN PROGRESS (Specialist gate partially implemented)
